// react 첨이여서 첨하면서 알게된거 아래 끄적끄적(백지에서 쓴거라서 매우 이상할 수 있음 정확한 명칭 등)

var 은 아무타입이나여서 .ts 타입을 지정해주는 거에서 퇴화 안씀

const 상수, let 변수

useState 에서 const쓰는 이유는 변수 재할당 막아줘서(의도하지않은 리렌더링 과정에서 사용될까봐)(안전장치인듯)

useState는 왼쪽 변수 오른쪽 변수에 값 할당 함수 usetState 괄호 안은 초기변수값 변수 바뀌면 해당 값이 사용된 dom요소만 수정 

[React 개발의 표준 및 커뮤니티 컨벤션] const test = () => {} 형태의 화살표 함수 권장

onClick에서 사용함수가 있을 경우 초반에 렌더링 되면서 해당 함수가 실행이 되는데 함수 내용안에 useState 리렌더링 하는게 있음 안됌(무한루프 가능성으로) 
그래서 onClick={() => test()} 이렇게 한번 감싸야함. useState변수 없음 안해도 되긴하는데 권장인데 .ts는 무조건인듯 걍 하는게 정밴듯

import 해서 가져올게 파일에 type값만 있다면 import type { User, } from '../../packages/shared-types/user.ts'; 이렇게 해야댐

Hook: state(상태)나 생명주기 같은 기능을 낚아채서(Hook) 쓸 수 있게 해주는 함수  (hook을 순서대로 기억함 위에서 아래로 그래서 최상단에 해야함)
Hook은 최상단에 써야함, 함수안에 사용하면 안됌

localstorage는 영구적(수동삭제전까지 데이터 유지) sessionStorage는 브라우저 탭 닫히면 데이터 삭제(탭/창마다 독립적인 데이터)

useEffect 컴포넌트가 처음 로드될 때 실행함(화면 다 그려진후 렌더링 후 ) 그래서 사용자 데이터 가져오는 코드를 보통 넣음

속성값 빼고 다 작은 따옴표 속성만 "" 큰 따옴표임 다른건 작은 따옴표

변수/속성 이름: camelCase (예: userName, menuList, categoryId)
함수 이름: camelCase (예: handleCLick, fetchData)
컴포넌트 이름: PascalCase (예: MenuPage, Spinner)

db에서는 보통 속성 _ 언더스코어 케이스 snake_case
