var 은 아무타입이나여서 .ts 타입을 지정해주는 거에서 퇴화 안씀

const 상수, let 변수

useState 에서 const쓰는 이유는 변수 재할당 막아줘서(의도하지않은 리렌더링 과정에서 사용될까봐)(안전장치인듯)

useState는 왼쪽 변수 오른쪽 변수에 값 할당 함수 usetState 괄호 안은 초기변수값 변수 바뀌면 해당 값이 사용된 dom요소만 수정 

[React 개발의 표준 및 커뮤니티 컨벤션] const test = () => {} 형태의 화살표 함수 권장

onClick에서 사용함수가 있을 경우 초반에 렌더링 되면서 해당 함수가 실행이 되는데 함수 내용안에 useState 리렌더링 하는게 있음 안됌(무한루프 가능성으로) 
그래서 onClick={() => test()} 이렇게 한번 감싸야함. useState변수 없음 안해도 되긴하는데 권장인데 .ts는 무조건인듯 걍 하는게 정밴듯
